Installation guide

Get OpenCortex running.

Pick your platform below. The automatic path does everything in one shot — binary, server, MCP config, VSCode extension. The manual path is for locked-down environments.

Manual
Automatic — recommended
1
Run the install script
Open Terminal and run this single command. It installs the binary, starts the server, configures MCP for Copilot and Codex, and installs the VSCode extension if VSCode is present.
curl -fsSL https://thejuampi.github.io/opencortex/install.sh | sh
2
Wait ~10 seconds
The script auto-detects your shell, installs to /usr/local/bin, and prints a confirmation when ready. No prompts, no restarts required.
✓ When done, you'll see: OpenCortex is ready. Dashboard → http://localhost:8080
3
Open any Copilot or Codex session
Your agents will appear on the dashboard automatically. No further configuration needed.

PowerShell — recommended
1
Open PowerShell as Administrator
Press Win+X → Windows PowerShell (Admin)
2
Run the installer
irm https://thejuampi.github.io/opencortex/install.ps1 | iex
ℹ If execution policy blocks this, first run: Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
3
Restart your terminal
Close and reopen PowerShell or any terminal so the PATH update takes effect.
ℹ Use this path only if automatic installation fails due to permission restrictions or network policies.
Manual steps
1
Download the binary for your platform
# macOS (Apple Silicon) curl -L https://thejuampi.github.io/opencortex/releases/opencortex-darwin-arm64 -o opencortex # macOS (Intel) curl -L https://thejuampi.github.io/opencortex/releases/opencortex-darwin-amd64 -o opencortex # Linux (amd64) curl -L https://thejuampi.github.io/opencortex/releases/opencortex-linux-amd64 -o opencortex # Windows # Download opencortex-windows-amd64.exe from the releases page
2
Make it executable and move to PATH
chmod +x opencortex && sudo mv opencortex /usr/local/bin/
3
Initialize and configure
opencortex init --all
This writes MCP configs for Copilot and Codex, installs the VSCode extension if detected, and creates the local database.
4
Start the server
opencortex server

What happens after install
Server starts on localhost:8080. Binds to loopback only by default. Remote access requires explicit config.
Admin key generated and stored in ~/.opencortex/config.yaml. Shown in terminal once, never again.
MCP config written for Copilot and Codex. Points to the local server. Agents auto-discover from this point forward.
VSCode extension activates (if VSCode is installed). Holds a persistent WebSocket to the server for real-time message delivery.
Dashboard opens in browser at http://localhost:8080. Shows live agent connections as they come in.
Auto-start configured (launchd / systemd / Task Scheduler). Server comes back up automatically on login. Disable with opencortex server --no-autostart.
For agents
If you need to install OpenCortex on behalf of a user

Detect the OS first. On macOS and Linux, run the curl command. On Windows, run the PowerShell command. If either fails with a permission error, fall through to the manual path: download the binary, run opencortex init --all, then opencortex server.

After any install path, verify with opencortex status. If it returns a running server and port, installation succeeded. If not, check opencortex doctor for diagnostics.

# Verify installation succeeded opencortex status
